# Button 主题绑定 — 盘点与最少设置项

> 设置 Tab 右侧面板的设计依据。  
> 目标：**不按实例、不按「每个状态各配一遍」**，用最少项覆盖 TDesign Button 全部变体。

---

## 〇、根本约束：能设什么，取决于组件用了什么变量

**设置页能改样式，是因为组件样式走的是全局 CSS 变量（`--td-*`、`--flow-*` 等）。**

| 规则 | 含义 |
|------|------|
| **用了才能设** | 设置项 ↔ 组件内实际引用的变量一一对应；没引用的属性不出现在面板里 |
| **设了必生效** | 改变量 → 所有读该变量的实例一起变（填充 / 描边 / 幽灵 / 文字按钮同理） |
| **形态不增项** | 文字按钮、幽灵、描边、虚线填充 — 组件内部只是**同一批变量**的不同组合方式，**按钮级统一设置即可覆盖**，不必为每种 variant 各做一栏 |

因此：

- 圆角、边框宽、动效、字体等 **按钮统一项** 写进 `:root` 后，描边钮的边框、幽灵钮的字色、填充钮的底，凡是用到这些 token 的都会变。
- 语义色块只绑 **色板里组件会读的颜色变量**（主色 / hover / light 等），不绑「描边按钮专用背景」这种组件根本不读的虚构项。
- 面板字段应来自 **Button 样式表 / TDesign token 清单**，而不是来自 UI 上能点到的每一颗按钮。

## 一、Button 全量内容（要覆盖什么）

### 1. 语义类型 `theme`（5 种）

| 语义 | 典型用途 | 色板来源 |
|------|----------|----------|
| 常规 Primary | 主操作 | `--td-brand-color` |
| 成功 Success | 确认、完成 | `--td-success-color` |
| 警告 Warning | 警示操作 | `--td-warning-color` |
| 危险 Danger | 删除、不可逆 | `--td-error-color` |
| 灰色 Default | 次要、取消 | 灰阶 / 组件背景 |

### 2. 形态 `variant`（4 种）— **不能和语义合并成「类型」**

同一语义下，外观规则完全不同：

| 形态 | 默认底 | 边框 | 文字 | 说明 |
|------|--------|------|------|------|
| **填充** base | 语义主色 | 无或同色 | 衬色（常白） | 实心按钮 |
| **描边** outline | 透明 | 语义主色 | 语义主色 | 悬停常铺浅色底 |
| **虚线** dashed | 同描边 | 虚线 | 语义主色 | 弱强调 |
| **幽灵** text | 透明 | 无 | 语义主色 | 最轻量 |

若设置项只按「成功 / 危险」分，会像要给「描边钮单独配背景」——但组件读变量时幽灵/描边根本**不用**填充底那一项，所以不该在设置里出现。

形态差异在 **组件实现层**用同一套 CSS 变量做组合（见 §三 配方），**设置层不增字段**。

### 3. 尺寸 `size`（3 种）

大 / 中 / 小 — 主要差在**内边距与字号**，圆角一般跟组件统一。

### 4. 形状 `shape`（可选）

矩形 / 圆角 / 圆形 — 通常只吃**统一圆角**或固定规则，不单绑。

### 5. 交互态（你提到的 3 种 + 扩展）

| 交互态 | 用户感知 |
|--------|----------|
| **悬浮** hover | 颜色略变、可有过渡 |
| **按下** active | 更深/更实 |
| **聚焦** focus | 焦点环（常与 outline 叠加） |

扩展（可先不放进设置面板）：`disabled`、`loading`。

### 6. 变体交叉（为何不能逐项绑）

```
5 语义 × 4 形态 × 3 交互态 × 3 尺寸 ≈ 180 种组合
```

逐项绑颜色会爆炸。必须拆成：**公共规则 + 语义色 + 形态配方 + 色板派生交互色**。

## 二、公共项 vs 差异项

### 按钮统一项（对 Button **全部**语义、形态、尺寸、交互态生效）

> 不叫「公共设置」这个 UI 名，而是**组件级统一绑定**；运行时所有按钮实例都读这一套。  
> **填充 / 描边 / 虚线 / 文字（幽灵）按钮共用这些变量**，故统一项即可覆盖，无需按形态再拆设置区。

| 项 | 说明 | 设置数量 |
|----|------|----------|
| 圆角 | 所有形态共用（圆形按钮可走 shape 规则） | 1 |
| 边框宽度 | 描边/虚线用；填充可为 0 | 1 |
| 边框线型 | 虚线 vs 实线由 **形态** 决定，不单独绑 | 0（规则） |
| 表面效果 | 阴影等（幽灵/描边常为 none） | 1 |
| 动效 | 过渡时长/缓动（悬浮、按下） | 1 |
| 字体 | 大 / 中 / 小 各一套命名样式 | 3 |
| 交互态字体（可选） | 如按下改用更小/更淡字重 | 0～3 |

**公共项合计：约 7～10 项。**

### 差异项（随语义 / 形态 / 交互变化）

| 差异来源 | 变什么 | 能否单独做设置 Tab？ |
|----------|--------|----------------------|
| **语义** | 主色及衍生色 | ✅ 只绑颜色 |
| **形态** | 底/边/字谁用主色、谁透明 | ❌ 组件内配方组合同一批变量，不绑 |
| **交互态** | 主色 → hover/active 色 | ⚠️ 见下文最少方案 |
| **尺寸** | 字号、padding | ✅ 字体 3 项（padding 可走尺寸尺度 token） |

## 三、颜色：语义里有什么、形态怎么「吃」语义色

### 每种语义只需维护的**颜色角色**（建议 3 个引用，不是 9 个）

| 角色 | 用途 | 示例引用 |
|------|------|----------|
| **主色** `main` | 填充底、描边边、幽灵字 | `--td-brand-color` |
| **衬色** `on-main` | 填充按钮上的字 | `#ffffff` |
| **浅底** `main-light`（可选） | 描边/幽灵 hover 铺底 | `--td-brand-color-light` |

悬浮/按下/禁用：优先**跟色板已有** `--td-*-hover` / `--td-*-active` / `--td-*-disabled` 走，不再在设置里 ×5 语义各配 3 态。

### 形态 = 配方（零设置项）

```
填充：  bg = main,     text = on-main,  border = none
描边：  bg = transparent, border = main, text = main,  hover.bg = main-light
虚线：  同描边 + border-style: dashed
幽灵：  bg = transparent, border = none, text = main, hover 可 text = main-hover
```

设置面板里**不出现**「描边按钮的成功色」这种项，只出现「成功语义的主色/衬色」。

## 四、最少设置项方案（推荐）

分 **两块 UI**（名称待定，不用「公共设置」）：

### 块 A — 按钮统一（组件级，全状态生效）

| # | 设置项 |
|---|--------|
| 1 | 圆角（尺寸尺度 token） |
| 2 | 边框宽度 |
| 3 | 效果预设 |
| 4 | 动效预设 |
| 5～7 | 字体预设：大 / 中 / 小 |

可选：交互态字体覆盖（悬浮 / 按下 / 聚焦）各 1 项，默认「沿用尺寸字体」。

### 块 B — 语义颜色（5 种 × 仅颜色）

每种语义 **2～3 个色板变量引用**：

| 语义 | 主色 | 衬色 | 浅底（可选） |
|------|------|------|--------------|
| 常规 | `--td-brand-color` | 白 | `--td-brand-color-light` |
| 成功 | `--td-success-color` | 白 | `--td-success-color-light` |
| … | … | … | … |

**不绑**：形态、尺寸、hover/active 的逐格颜色（交给色板 + 配方）。

### 块 C — 交互态（3 种）— 二选一

| 方案 | 设置项数 | 做法 |
|------|----------|------|
| **C1 推荐** | **0** | 交互色全部由色板 `--td-{语义}-hover/active` 自动映射 |
| **C2** | **3** | 按钮级统一：悬浮规则 / 按下规则 / 聚焦规则（各选一种 palette 变量或亮度%，作用于所有语义） |

你提的「三种交互态」更适合 **C1 + 聚焦环用统一边框色/效果**；若设计要全局改「按下变深多少」，用 C2 三条规则即可，**不要** 5×3×3 网格。

---

## 五、设置项总数（对比）

| 方案 | 大约项数 | 说明 |
|------|----------|------|
| 现状（每语义×每交互×bg/text/border） | 45+ | 过多，且重复 |
| **推荐最少** | **12～15** | 统一 7 + 语义 5×1主色（衬色规则固定）或 5×2 |
| 加交互规则 C2 | +3 | 共 15～18 |

## 六、与左栏预览的对应

| 预览分组 | 验证什么 |
|----------|----------|
| Button → 颜色 | 5 语义 × 填充 + 描边 + 幽灵 |
| Button → 动效 | 统一动效预设 |
| Button → 尺寸与形态 | 统一字体 + 圆角 |

右侧面板选中 **Button** 时，只展示块 A + 块 B（+ 可选 C2），不按预览里的每一颗按钮各一行设置。
